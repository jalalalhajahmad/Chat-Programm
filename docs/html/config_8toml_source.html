2# @file config.toml
3# @brief Configuration file for SLCP clients.
4#
5# This configuration file defines the list of available clients for the
6# SLCP peer-to-peer chat application. Each client section contains
7# the networking and UI preferences used to start an instance.
8#
9# @section format_sec Format
10# The file uses the TOML format and contains a list of [[clients]] tables.
11#
12# @section fields_sec Fields
13# - handle: Unique name/identifier for the client.
14# - port: List of two ports. First is the UDP port for peer communication,
15# second is typically used for image transfer (or a reserved fallback).
16# - whoisport: Broadcast port used for WHO and JOIN messages.
17# - autoreply: Message sent automatically when the user is AFK.
18# - away: Boolean flag indicating whether the user starts in AFK mode.
19# - imagepath: Path to the local folder where received images will be stored.
20#
21# @note All clients share the same whoisport for discovery purposes.
22
23[[clients]]
