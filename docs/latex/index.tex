A decentralized, peer-\/to-\/peer chat application developed in Python for the {\bfseries{Betriebssysteme und Rechnernetze}} (Operating Systems and Computer Networks) course. This project implements a custom protocol called SLCP (Simple Local Chat Protocol) to demonstrate real-\/time communication over local networks without relying on central servers.\hypertarget{index_autotoc_md1}{}\doxysection{\texorpdfstring{Project Description}{Project Description}}\label{index_autotoc_md1}
The project aims to simulate a real-\/world decentralized chat application that supports direct peer-\/to-\/peer communication without relying on a central server. By using both UDP and TCP sockets, the system enables\+:
\begin{DoxyItemize}
\item Dynamic discovery of other peers
\item Text-\/based chat communication
\item Image file sharing
\item Away-\/from-\/keyboard (AFK) autoreply functionality
\item Two fully-\/featured interfaces\+: a command-\/line interface (CLI) and a graphical user interface (GUI)
\end{DoxyItemize}

This system mirrors the principles of modern decentralized applications where reliability, independence, and peer autonomy are key. The communication protocol (SLCP) was developed specifically for this project, ensuring messages are structured, lightweight, and interpretable.\hypertarget{index_autotoc_md2}{}\doxysubsection{\texorpdfstring{Challenges Faced}{Challenges Faced}}\label{index_autotoc_md2}
Developing a decentralized peer-\/to-\/peer application came with several challenges\+:
\begin{DoxyItemize}
\item {\bfseries{Concurrency Management\+:}} Synchronizing multiple processes (network, discovery, GUI, CLI) required careful use of multiprocessing, threading, and pipes.
\item {\bfseries{Peer Discovery\+:}} Broadcasting JOIN and WHO messages while avoiding duplicate peers or stale data demanded a reliable protocol logic.
\item {\bfseries{Image Transfer\+:}} Transferring large binary files over TCP while signaling over UDP required a clean and fail-\/safe handshaking mechanism.
\item {\bfseries{Interface Parity\+:}} Maintaining full feature parity between CLI and GUI was non-\/trivial, especially with user interactions.
\item {\bfseries{AFK Logic\+:}} Automatically replying with a custom message while avoiding spam loops required careful state tracking.
\item {\bfseries{Cross-\/platform Compatibility\+:}} Ensuring the system runs reliably on Windows, mac\+OS, and Linux involved path handling, port availability checks, and encoding robustness.
\end{DoxyItemize}

\hypertarget{index_autotoc_md33}{}\doxysection{\texorpdfstring{Technologies}{Technologies}}\label{index_autotoc_md33}

\begin{DoxyItemize}
\item {\bfseries{Python 3.\+10+}} ~\newline
 The main programming language used for the entire application. Python\textquotesingle{}s high-\/level syntax and standard library make it ideal for rapid development and academic projects.
\item {\bfseries{Py\+Qt5}} ~\newline
 Used to create the graphical user interface (GUI). Offers support for event-\/driven programming through signals and slots, as well as cross-\/platform compatibility.
\item {\bfseries{TOML}} ~\newline
 A minimal and human-\/readable configuration file format used to define client settings such as handles, ports, autoreplies, and image directories.
\item {\bfseries{Socket Programming (UDP/\+TCP)}} ~\newline
 Enables real-\/time communication between peers. UDP is used for message broadcasting ({\ttfamily JOIN}, {\ttfamily WHO}, {\ttfamily MSG}, etc.), while TCP is used for transferring binary image files.
\item {\bfseries{Doxygen}} ~\newline
 Automatically generates HTML-\/based technical documentation from docstrings and markdown files such as {\ttfamily \doxylink{_r_e_a_d_m_e_8md}{README.\+md}}.
\item {\bfseries{Multiprocessing Pipes ({\ttfamily multiprocessing.\+Pipe})}} ~\newline
 Used for inter-\/process communication (IPC) between the GUI/\+CLI and the network process. Allows sending structured messages like {\ttfamily (MSG, handle, message)} through unidirectional or bidirectional channels.
\item {\bfseries{Threads ({\ttfamily threading.\+Thread})}} ~\newline
 Used for non-\/blocking background tasks like periodic broadcasting ({\ttfamily JOIN}, {\ttfamily WHO}) and image transfers. Ensures responsiveness of the GUI and CLI.
\item {\bfseries{Shared Memory (via {\ttfamily multiprocessing} state)}} ~\newline
 Certain configuration states (e.\+g., AFK status) and message buffers are indirectly synchronized across processes by sharing references during process creation.
\item {\bfseries{QDark\+Style}} ~\newline
 A ready-\/made dark mode theme applied to the Py\+Qt5 interface for improved aesthetics and readability.
\end{DoxyItemize}
